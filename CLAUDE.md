# CLAUDE.md

## ワークフロールール

### ブランチ戦略

- 作業ブランチ
  - feature/*: 機能開発
  - chore/*: コメントなどの軽微な変更
  - doc/*: ドキュメント更新
- 開発ブランチ
  - develop
- 最新ブランチ
  - main

### PRを作成するとき
PRを作成する際は、必ず以下の手順を守ること：

1. コードの変更を完了する
2. `gh pr create` でPRを作成する
3. **PRを作成したら、必ず直後に `/review` コマンドを実行してレビューを行い、PRへコメントを入れる**
4. レビュー結果で指摘された問題があれば修正してから完了とする

この手順を省略しないこと。
